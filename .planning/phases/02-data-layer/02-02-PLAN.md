# Plan 2-2: QC Input Form UI

## Objective
Build a dynamic, single-page input form for QC staff to register new incoming vehicles. The form will conditionally display metric input fields depending on whether the selected material is CPO, PK, or Cangkang.

## Code Patterns (Model Resilience)
Conventions:
- Client Components: Form UI should use `"use client"` so we can dynamically toggle input fields based on the selected dropdown for Material Type.
- Server Actions: Handling database insertions securely.
