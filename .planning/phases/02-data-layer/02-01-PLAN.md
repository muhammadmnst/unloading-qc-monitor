# Plan 2-1: VehicleLog Database Schema

## Objective
Implement a unified `VehicleLog` table using Prisma. This table will support conditional metrics for CPO, PK, and Cangkang, and carefully track timestamps across the entire vehicle status lifecycle (Pending, Bongkar, Cancel, Completed).

## Code Patterns (Model Resilience)
Conventions:
- Prisma Schema: Centralized enums for `MaterialType` and `VehicleStatus`.
- Timestamps: Store specific state transitions (`pendingAt`, `bongkarAt`, etc.) to facilitate R10 tracking. Nullable relations `?` used strictly per material.
