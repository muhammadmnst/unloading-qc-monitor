# Plan 3-1: Dashboard Data Layer & Actions

## Objective
Implement severe-side actions to fetch dashboard logs and mutate vehicle states based on user interaction ("Mulai Bongkar", "Cancel", "Completed"). 

## Code Patterns
Conventions:
- Action Handlers: Use Next.js `"use server"` actions for data mutations (`updateStatus`) and data fetching (`getDashboardLogs`).

## Tasks

  <action>
    1. Create `getDashboardLogs(materialType: string, showCompleted: boolean)`.
       - Query Prisma for `VehicleLog` where `materialType` matches.
       - If `!showCompleted`, exclude status `COMPLETED`.
       - Sort: PENDING always at the bottom, mapping sorting logic correctly (e.g. order by status descending or conditional).

       - If `newStatus === 'CANCEL'`, set `cancelAt = new Date()`.
       - If `newStatus === 'COMPLETED'`, set `completedAt = new Date()`.
       - Update the log in Prisma.
  </action>
