# Plan 1-4: Admin User Management UI

## Objective
Membuat kontrol antarmuka admin berbentuk tabel (Data Tables Shadcn) yang melist semua user. Memungkinkan penambahan user QC/Ops/Admin, Update Role, dan Hapus (Delete).

## Code Patterns (Model Resilience)
Conventions:
- Server Actions Next.js untuk merespon operasi `createUser` & `deleteUser`.
- `bcryptjs.hash()` di dalam Create User action untuk meracik raw password yang disubmit admin.
