# Plan 1-3: Autentikasi (NextAuth) & RBAC

## Objective
Mengunci aplikasi menggunakan implementasi NextAuth Credentials Provider. Memilah validasi Role (Admin, QC, Ops) langsung dari edge runtime middleware, dan mengamankan hashing password.

## Code Patterns (Model Resilience)
Conventions:
- Penggunaan library `bcryptjs` untuk validasi `password` plain vs hash.

## Tasks

<task type="auto">
  <name>Setup Auth.js & Login UI</name>
  <files>auth.ts, middleware.ts, src/app/api/auth/[...nextauth]/route.ts</files>
  <action>
    1. Instal `next-auth@beta`, `bcryptjs`, dan `@types/bcryptjs`.
    2. Buat `auth.ts` dengan Provider `Credentials`. Validasi username & check bcrypt hash terhadap DB Prisma. Custom session callback untuk menyusupkan `role`.
    3. Daftarkan API route `route.ts`.
    4. Buat `middleware.ts` untuk memblokir rute selain `/login` dan rute TV spesifik (`/tv`). Atur route protection berbasis `request.auth.role` sehingga `/admin` hanya "ADMIN".
    5. Buat halaman `/login` dengan Next.js Server Actions untuk menerima input kredensial ganda (Username, Password).
  </action>
  <verify>Akses rute yang diprotect harus diredirect 302 ke `/login`.</verify>
  <done>Sistem RBAC dan middleware login berfungsi mengamankan portal.</done>
</task>

## Must-Haves
- Middleware berfungsi pada skema rute `/admin`, `/dashboard`, namun membebaskan `/tv`.
- Autentikasi berjalan *stateless* murni dengan JWT untuk simplisitas.
