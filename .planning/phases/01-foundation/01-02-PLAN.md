# Plan 1-2: Skema Database & Prisma ORM

## Objective
Melakukan setup `Prisma` dan mendefinisikan skema model awal, yaitu tabel *User* dengan Role Enum (Admin, QC, Ops).

## Code Patterns (Model Resilience)
Conventions:
- Prisma best practice: Singleton client di `src/lib/prisma.ts`.

## Tasks

<task type="auto">
  <name>Init Prisma & User Model</name>
  <files>prisma/schema.prisma, src/lib/prisma.ts</files>
  <action>
    1. Install `prisma` dan `@prisma/client`.
    2. Setup berkas `.env` menuju koneksi string Docker (`postgresql://postgres:postgres@localhost:5432/unloading_qc?schema=public`).
    3. Definisikan model `User`: `id` (UUID), `username` (String @unique), `password` (String), `role` (Enum: ADMIN, QC, OPERATIONAL), `createdAt`, `updatedAt` di `schema.prisma`.
    4. Buat file koneksi database terpusat pada `src/lib/prisma.ts`.
    5. Jalankan migrasi baselinenya `npx prisma migrate dev --name init`.
  </action>
  <verify>npx prisma studio --browser none (Cek apakah prisma mampu membaca DB)</verify>
  <done>Tabel terinisiasi di Postgres lokal dan `prisma` di proyek Next ter-generate valid.</done>
</task>
